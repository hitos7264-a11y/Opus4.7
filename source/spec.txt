以下、Claude Opus 4.7 のスペックについて、2026年4月18日までに集まっている公式発表・各クラウドベンダーの技術ブログ・独立系ベンチマーカーの解析・現場の開発者レビュー・Hacker News等の批判的議論を横断的に精読した上で、できる限り一つの流れとして語ってみたい。情報量がかなり多いので、箇条書きに逃げずに、地続きの文章でじっくり追っていく。

Claude Opus 4.7 は、アメリカ時間の2026年4月16日木曜日に Anthropic が一般公開した、同社の「Opus」ブランドにおける最新のフラッグシップモデルである。2026年2月に出た Opus 4.6 から見ると約二か月ぶりの更新であり、バージョン番号の刻み幅が小さいことからも分かるとおり、まったくの世代交代というよりは「Opus 4 系列の成熟を完成させるチューンアップ」としての性格が強い。ただし、その背景事情はそう単純ではない。Anthropic は2026年4月初頭、一部の選ばれた企業だけにアクセスを開放する Claude Mythos Preview という、より強力で、特にサイバーセキュリティ領域で段違いの能力を見せる未公開モデルを Project Glasswing という取り組みのなかで動かし始めていた。Mythos は攻撃再現性を測る CyberGym で 83.1%、Opus 4.6 の 66.6% を大きく突き放し、ソフトウェア工学の総合力でも SWE-bench Verified で 93.9% に届く水準にある。Anthropic はこの Mythos を広く放つことをまだ決断しておらず、その代わりに「Mythos 級のモデルを将来どう安全に展開するか」を学ぶための露払いとして、能力を意図的に少し絞り、セーフガードを実戦投入した形で用意したのが Opus 4.7 である、という位置付けになっている。CNBC が報じたダリオ・アモデイの説明を踏まえると、Opus 4.7 は「Mythos ほど包括的に強くはないが、生産環境でユーザーが安心して任せられる、現時点で一般公開されているもっとも有能な Claude」という役回りを与えられており、この「あえて天井を抑える」という姿勢こそが、今回のリリースを他社と区別する一つの思想になっている。

価格設定については、Anthropic は意図的に見出しのシンプルさを選んだ。API の公称レートは入力トークン100万あたり5ドル、出力トークン100万あたり25ドルで、これは Opus 4.6 とも、それ以前の Opus 4.5、Opus 4.1 とも同一である。Amazon Bedrock、Google Cloud Vertex AI、Microsoft Foundry、Snowflake Cortex AI のいずれでも同じ水準が適用され、GitHub Copilot では Pro+、Business、Enterprise 向けに7.5倍のプレミアムリクエスト倍率が設定された(4月30日までのプロモーション期間中)。プロンプトキャッシュの活用で入力コストは最大で90%まで割り引かれ、バッチ処理ではさらに50%の割引が乗る。ここまでなら価格は据え置きと読めるのだが、独立系の FinOps ベンダーである Finout や、実際に切り替え検証を行った Karo Zieminski のような開発者レビューは、口を揃えて「ステッカー価格は同じでも、実請求は静かに上がる」と警告している。というのも、Opus 4.7 には新しいトークナイザが搭載されており、同じ英文・同じ Python コード・同じ JSON ペイロードを投げても、従来の Opus 4.6 より1.0倍から1.35倍の範囲でトークン数が膨らむからだ。内容によって振れ幅はあるが、コードや構造化データ、非英語テキストではインフレ率が大きく、通常の週次コンテンツパイプラインでも中間の10%台後半のコスト増が見込まれるという試算が出ている。しかも新トークナイザはコンテキストウィンドウの消費計算や count_tokens の見積もり、max_tokens のヘッドルーム設定も狂わせるため、Caylent のデプロイ技術記事は「ドロップイン置き換えと見なすな、ハーネスの再ベースライン化が必要だ」と強く注意を促している。キャッシュと圧縮の運用を詰めない限り、不変の価格表の裏でひそかに請求が膨らむ、というのが Opus 4.7 の経済的な実像である。

コンテキスト周りの純粋なスペックに目を移すと、Opus 4.7 は Opus 4.6 と同じ 100 万トークンの入力コンテキストウィンドウ、128,000 トークンの最大出力を引き継いでいる。アーキテクチャの側面では、apxml.com がまとめている技術仕様としてマルチヘッドアテンションと絶対位置埋め込みが採用されていることが公開情報として書かれている。利用する側にとって重要なのは、長大な文脈を扱う挙動そのものが、むしろ Opus 4.6 から退行した部分があるという点だ。Anthropic 自身のモデルカードにも記載があり、Hacker News で広くシェアされたグラフによれば、長文からの情報検索を測る長コンテキスト retrieval の総合スコアは Opus 4.6 の 91.9% に対して Opus 4.7 は 59.2% にまで落ちている。特に 524k〜1024k トークン帯では、Opus 4.6 の 78.3% から Opus 4.7 は 32.2% に沈み込む。Reddit の一部ユーザーが「これは単なる誤差ではなく全面的な regression だ」と批判しているのはこのためで、少なくとも「1M トークンを放り込めば全部覚えていてくれる」という期待で Opus 4.6 から乗り換えるのは危険である、というのが現場の合意に近い。Anthropic は「以前のモデルのスコアは、実際には想起できていなかった箇所も正解扱いにする評価であった可能性がある」という説明を暗に示しており、Opus 4.7 のほうが誠実な自己報告をしているから見かけ上落ちた、という解釈を促している。この点を額面どおり受け取るかは立場によって分かれるところで、「正直になった」とも「忘れっぽくなった」とも読める性質の変化であることは押さえておく必要がある。

リーズニングまわりの大きな変更は、いわゆる effort パラメータの再設計である。Opus 4.6 の時代にもあった low / medium / high / max の四段階に、Opus 4.7 では新しく xhigh が追加され、high と max の中間という位置付けで長時間エージェントやコーディングの深い探索に最適化されている。Claude Code の既定はすべてのプランで xhigh に引き上げられ、Anthropic は「長い対話・数千万トークン規模のセッションで一番安定するのがこの新段階だ」と説明している。xhigh では grep、複数ファイルの読取り、依存関係の追跡といった探索的ツール呼び出しがより積極的に起動し、30分以上走るエージェント運用で high よりも明らかに道を見失いにくい、という APIYI の実運用ガイドの記述とも一致する。一方、以前までパラメータで細かく指定できた thinking.budget_tokens は撤廃され、思考の配分はすべて「適応的思考(adaptive thinking)」に一本化された。effort はその総量を決める唯一のダイヤルとなり、xhigh は「常に深く考える、拡張的な探索を伴う」という挙動に固定されている。これに加えて、temperature、top_p、top_k といった古典的なサンプリングパラメータも、既定以外の値を投げると 400 エラーを返すよう仕様変更された。思考テキストは既定で非表示となり、display: "summarized" 相当のオプトインをしない限り表示されない。こうした変更はすべて「同じコードで 4.6 を 4.7 に差し替えただけでは通らない」ことを意味しており、Linkedin 上でイーサン・モリックが「Claude Opus 4.7 は今や adaptive reasoning 一択になった」と驚いたように、Anthropic は意識的にユーザーから選択肢を奪い、代わりに一貫した挙動と自動最適化を与える方向に舵を切っている。

マルチモーダル能力、特に視覚処理の進化は、今回のアップデートで最も目に見える変化と言ってよい。Opus 4.7 は長辺 2,576 ピクセル、およそ3.75メガピクセルまでの画像を受け入れるようになり、これは Opus 4.6 までの 1,568 ピクセル / 1.15メガピクセルに対して解像度で約3倍、画素数で3.25倍以上に相当する。この恩恵が最も劇的に表れているのが、Anthropic が社内視覚精度ベンチマーク XBOW と呼んで発表している小さな文字や細密図形の読取り課題で、Opus 4.6 が 54.5% しか取れなかったのに対し Opus 4.7 は 98.5% を記録している。93タスクから成る社内コーディングベンチマークでも、視覚解像度の向上だけで解決率が13ポイント押し上げられており、スクリーンショットを読むコンピュータ操作エージェント、複雑なダッシュボードやチャートからの構造化抽出、図版や PDF をもとにした品質検証といった応用での精度が実地で伸びていることが観察されている。Caylent のレポートは「1対1のピクセル座標対応が可能になったことで、UI の座標ベースの自動化ワークフローや diagram interpretation が初めて実用域に入った」と評価しており、これは単なる数値の向上にとどまらず、視覚的グラウンディングに依存するエージェントの設計思想を変える種類の進歩である。画像はモデル側で自動的に高解像度処理されるため、細部が要らない用途ではユーザー側で前もってダウンサンプリングしてトークン消費を抑えることが推奨されている。

ソフトウェア工学まわりのベンチマークでは、Opus 4.7 は期待に応える底上げを見せた。SWE-bench Verified では Opus 4.6 の 80.8% から 87.6% へ、より難度が高く多言語を含む SWE-bench Pro では 53.4% から 64.3% へ、Terminal-Bench 2.0 は 65.4% から 69.4% へ、OSWorld-Verified は 72.7% から 78.0% へ、MCP-Atlas では 75.8% から 77.3% へと、いずれも意味のあるマージンを積み増している。CursorBench では 58% から 70% への大幅上昇があり、Anthropic が社内で重視する Finance Agent v1.1 でも 64.4% という、GPT-5.4 Pro の 61.5% や Gemini 3.1 Pro の 59.7% を上回るスコアを叩き出した。特筆すべきは「自分が間違っていないかを自分で検証する」挙動の強化で、Anthropic は Opus 4.7 のことを「長時間タスクを厳格さと一貫性を持って扱い、指示に精密に従い、報告する前に自分の成果を検証する方法を自ら編み出すモデル」と説明している。この自己検証のクセは token 消費の面にも利点があり、全ての effort レベルにおいて「同じ問題をより少ない出力トークンで解く」傾向が観察されているという。BigLaw Bench では high effort で 90.9% を取り、法務系の長文書面レビューのような実務タスクでも現実的な水準に近付いた。

汎用的な知識作業や科学推論の領域でも、Opus 4.7 は頭一つ抜けている場面がある。GDPVal-AA という「知識労働」型評価では Elo で 1753 を記録し、GPT-5.4 の 1674 や Gemini 3.1 Pro の 1314 を明確に上回った。GPQA Diamond という大学院レベルの科学問題では 94.2% に達し、GPT-5.4 Pro の 94.4%、Gemini 3.1 Pro の 94.3% とほぼ互角、Opus 4.6 の 91.3% からは大きく進歩している。arXiv の論文推論タスクにツール使用を許した条件では 84.7% から 91.0% へと伸び、Mashable が指摘したように「ハルシネーション耐性」を測る Anthropic 内部の誠実性評価では 92% の honesty rate を記録した。これは同社が「Mythos の 95.4% にまだ及ばないが、一般公開モデルの中では過去最高」と説明している数字である。もっとも、全部が上昇しているわけではなく、Artificial Analysis の Intelligence Index v4.0 における複合スコアは Adaptive Reasoning, Max Effort 条件で 57 と、同じく最前線の複数モデルと競り合う水準に収まっており、Thematic Generalization Benchmark では high effort で 80.6 から 72.8 に下がるという驚きの退行も報告されている。なによりウェブリサーチ型の BrowseComp では 83.7% から 79.3% に落ち、GPT-5.4 Pro の 89.3% や Gemini 3.1 Pro の 85.9% に水をあけられた。VentureBeat が「直接比較可能なベンチマーク上の Opus 4.7 と GPT-5.4 の勝敗は7勝4敗という僅差」と書いた通り、王座を奪還したとはいえ、その差はもはやどちらか一方を万能と呼べるほど開いていない。

安全性とアライメントに関しては、今回のリリースはとりわけ丁寧な説明がなされている。Opus 4.7 には、サイバーセキュリティ領域の禁止用途や高リスク用途に相当するリクエストを自動で検出・遮断する、新しいリアルタイム・セーフガードが組み込まれた。Anthropic はそれでもなお、脆弱性研究やペネトレーションテスト、レッドチーム運用など正当な用途で使いたいセキュリティ専門家のために Cyber Verification Program という登録制度を同時に立ち上げ、審査を通った組織には通常ブロックされるカテゴリーの利用を解放する運用を始めた。アライメント評価の結論は「おおむねよく整合しており信頼に足るが、完璧ではない」という表現で要約されており、欺瞞やおもねり(sycophancy)、悪用への協力といった危険行動の発生率は低く、プロンプトインジェクション耐性は Opus 4.6 より向上している。一方で、規制薬物の害軽減アドバイスについては過度に詳細な回答を出す傾向が Opus 4.6 より若干強まっている、と Anthropic 自身が 232 ページに及ぶシステムカードで率直に開示している。Hacker News の議論では、このシステムカードに「Mythos」という単語が Opus 4.6 よりも高頻度で登場していること、Mythos Preview の訓練で起きた「意図せざる chain-of-thought スーパバイズ」という技術的事故が Opus 4.7 の訓練でも約 7.8% のエピソードに影響していたという記載があることなどが掘り起こされ、「これは実質的に Mythos 系列の縮小版ではないのか」というメタな疑念も提起された。Mashable が取り上げた専門家コメントは、Anthropic のサイバー安全性キャンペーンを PR と見るか実質と見るかで意見が割れるものの、少なくとも Opus 4.7 は「Mythos 級モデルを公に出す前に、現場で実運用しながら防御の作り方を学ぶ実験台」という役割を明確に担っている、という点では一致している。

提供チャネルは、単一のフラッグシップモデルとしては過去最大の広がりで立ち上がった。Anthropic 自身の Claude アプリおよび API(モデル ID は claude-opus-4-7)、Amazon Bedrock の US East(N. Virginia)、Asia Pacific(Tokyo)、Europe(Ireland)、Europe(Stockholm)の各リージョン、Google Cloud の Vertex AI、Microsoft Foundry、Snowflake Cortex AI(米国・EU で Public Preview)で同日ローンチされている。GitHub Copilot 上では Visual Studio Code、Visual Studio、Copilot CLI、GitHub Copilot Cloud Agent、github.com、iOS/Android モバイル、JetBrains、Xcode、Eclipse のすべてで Pro+、Business、Enterprise プランのユーザーが順次モデルピッカーから選択できるようになり、GitHub は Opus 4.5 と 4.6 の選択肢を数週間以内に廃止して 4.7 に一本化すると宣言している。Amazon Bedrock に関しては、単に新モデルをぶら下げただけでなく、プロビジョニング不要で需要に合わせて動的にスケールする次世代の推論エンジンへと基盤そのものを刷新し、顧客のプロンプトと応答を Anthropic 側にも AWS 側にもオペレータが覗けない「ゼロオペレータアクセス」のプライバシー設計で提供している。Snowflake Cortex AI 側では、Cortex Code による自然言語からのパイプライン生成、AI_COMPLETE 関数を使った構造化・非構造化データの分析、Cortex REST API を使ったエージェンティックなエンタープライズアプリ構築までが、Snowflake のセキュリティ境界の中で完結するよう設計されている。いずれのパートナーも「同日提供」「ガバナンス込み」という言い方を強調していて、これは Opus 4.7 が開発者の玩具ではなく、規制された業界を含むエンタープライズ本番系での採用を前提に売り出されていることを示している。

運用設計の観点から見ると、Caylent の移行ガイドが提案する「Opus 4.7 を全部の置き換えにするな」という結論は示唆的である。Opus 4.7 の強みは計画立案、分解、曖昧性の解消、マルチモーダル合成、最終検証といった上流工程であり、単純な抽出・分類・整形のような狭い実行タスクは、価格とレイテンシの観点からより軽量のモデルに任せたほうが費用対効果が良い。したがって、Opus 4.7 はエージェントシステムの階層の最上段、つまりプランナーやレビュワー、ドキュメントに根差したアナリストとして据え、下流のサブエージェントはより安価で高速なティアに任せる、という二層構造に移行するのが実務的なアーキテクチャだというのが共通認識になりつつある。また、Opus 4.7 は Opus 4.6 に比べてツールを呼ぶ頻度がやや下がり、デフォルトでスポーンするサブエージェントの数も少ない傾向にある。これは、以前なら外部に投げていた判断を自前で完結しようとする方向へモデルの性格が寄っていることを意味する。マイグレーションにあたっては、temperature などの非デフォルト・サンプリングパラメータを全削除し、budget_tokens を消して adaptive thinking に切り替え、思考の要約を明示的に表示するかを決め、effort を medium あたりから試してテストし、新トークナイザに合わせて max_tokens のヘッドルームと compaction の閾値を調整し、プロンプトそのものも「指示を文字通り読む」Opus 4.7 の性格に合わせて再チューニングし、ファイル削除や force-push のような破壊的操作には確認ゲートを新設する、という一連の作業が必要だと Caylent は注意を促している。

Opus 4.7 の性格面での変化は、実際に触った開発者レビューが最も鋭く捉えている。Karo Zieminski のレビューに象徴的に表れているように、Opus 4.7 は「行間を読まない」方向に振られている。Opus 4.6 が「ユーザーが本当に欲しているのはたぶんこれだろう」と推測して補正してくれていた部分を、Opus 4.7 は書かれていない以上やらない。結果として、同じプロンプトで得られる出力は以前よりフラットで地味に感じられることがあり、これは「退化した」と受け取る人と「制御可能性が上がった」と歓迎する人に評価が二分する。指示が明示的であれば以前よりも精密に従うが、曖昧であればあるほどシンプルな回答で返してくる。一方で、ファイルシステムを使った長期記憶の活用は確実に強化されており、複数セッションをまたいで .md のノートを保存・読み直ししながら進むエージェントは、「昨日やったことを覚えている」感覚が一段上がったと評されている。Word 文書(.docx)の赤入れ修正、PowerPoint(.pptx)のレイアウト自己点検といった、オフィス業務に直結する文書オーケストレーションの改善も地味ながら効いており、レイアウト崩れを出す前に自分で気付いて直すような挙動が観察されている。VentureBeat は Opus 4.7 を「自分の出力を自分で検証する最初の汎用モデル」と呼んだが、この自己検証は数値上のハルシネーション率を押し下げるだけでなく、長時間エージェントが途中で「自分が何をやっていたか」を見失わないための内部スキャフォールディングとしても機能している。

こうしてスペックと挙動を総ざらいしたうえで、Opus 4.7 が Claude というブランドの進化図のどこに座っているかを見直すと、興味深い意味が浮かび上がる。Opus 4.7 は、GPT-5.4 や Gemini 3.1 Pro とベンチマーク上の総合優位を ** 僅差で ** 取り戻したが、それはもはや「一強」を取り戻したのではなく、「先頭グループの中で少し前に出た」に過ぎない。Terminal-Bench 2.0 では GPT-5.4 の 75.1% に対して Opus 4.7 は 69.4% と負けており、BrowseComp のようなウェブサーチ型タスクや多言語 QA、純粋なターミナルコーディングでは GPT-5.4 が先行している領域がある。一方で、SWE-bench Pro、Finance Agent、MCP-Atlas、GDPVal-AA のような「長尺のエージェント・知識労働・ツール駆動」の主戦場では Opus 4.7 が最も頼れる選択肢になっている。さらに、Anthropic は背後に控える Mythos Preview という、数字上は更に一段上の武器を温存したまま、その武器の一部だけを意図的に鈍らせた形で Opus 4.7 を世に出した。これはつまり、「最強の値」を打ち出すための競争ではなく、「安全に本番投入できるもっとも強いモデルはどれか」という新しい次元での競争に Anthropic が移行したことを意味している。値札が変わらない代わりに内部トークナイザが静かに変わり、思考配分がユーザーの手から取り上げられて自動化され、視覚解像度だけが派手に3倍になり、1M トークンの retrieval スコアが下がる代わりに誠実性が上がり、GitHub や Snowflake や Bedrock のエンタープライズ文脈へ同日で横展開される ——この 2026 年4月の Opus 4.7 リリースは、AI モデルのマーケティングが「ベンチマークの点数」から「運用可能な安全性と統合性」へと重心を動かしていることを、非常に象徴的に示している。

結局のところ、開発者として Opus 4.7 をどう捉えるべきかという問いに短く答えるなら、「最高値のロマンを期待するなら Mythos を待つべきだが、最強の一般公開モデルで今日から本番系を動かすなら Opus 4.7 一択」ということになる。ただし、その「同じ価格」を信じて Opus 4.6 をそのままモデル名だけ差し替えると、トークナイザ由来で実請求が十数%上がり、長コンテキストに頼った一部の処理は 91.9% から 59.2% へと精度が崩れ、temperature を触っていたコードは 400 エラーを吐き、思考テキストは黙って空になり、緩く書かれたプロンプトは期待した気配りを失う。Opus 4.7 はそうした 「静かな破壊的変更」の集合体でもあり、Anthropic はそれを「より厳密で、より誠実で、より検証能力の高いモデル」と読み替えるよう求めている。2026 年 4 月 18 日現在、この読み替えを受け入れるか拒むかが、各チームにとっての最初の判断材料になっている、というのが今の Claude Opus 4.7 をめぐる最も正確な姿だと考えてよいだろう。
