長ctx retrieval の退行

Anthropic 自身のモデルカードに記載され、Hacker News で広くシェアされたグラフによれば、 長文からの情報検索を測る長コンテキスト retrieval の総合スコアは 91.9% → 59.2%。524k〜1024k tokens 帯では 78.3% → 32.2% まで沈む。

「1M 全部覚えている」期待は危険
r/ClaudeAI では「これは単なる誤差ではなく全面的な regression」という批判も。 少なくとも 524k 以上を前提にした RAG レス設計はやり直しが必要。
以前のモデルのスコアは、実際には想起できていなかった箇所も 正解扱いにする評価だった可能性がある。4.7 は誠実に自己報告している。 — Anthropic の自己弁護的説明（モデルカードより要約）

これを額面どおり受け取るかは立場によって分かれる。 「正直になった」とも「忘れっぽくなった」とも読める性質の変化である点は押さえておきたい。

新トークナイザ

同じ英文・同じ Python コード・同じ JSON を投げても、トークン数は 1.00〜1.35×に膨らむ。特にコード、構造化データ、非英語テキストでの インフレ率が高い。count_tokens の見積もりもコンテキストウィンドウ消費計算も 狂うため、ヘッドルームの再計算が必須。

影響項目再調整の要否
max_tokens の上限必須
compaction 閾値必須
コスト見積もり / 予算アラート必須
長文の chunking サイズ推奨
スライディングウィンドウの overlap推奨
