価格 & FinOps

「ステッカー価格は据え置き。でも実請求は静かに上がる」—— これが Opus 4.7 の 経済的な実像である。

GitHub Copilot のプレミアム倍率

4 月 30 日までのプロモ
7.5× premium request multiplier
GitHub Copilot では Pro+ / Business / Enterprise の各プランで、 Opus 4.7 を呼び出すときのプレミアムリクエスト倍率が 7.5× に設定されている。 これはプロモ期間中の暫定値で、VS Code / VS / CLI / Cloud Agent / github.com / iOS・Android / JetBrains / Xcode / Eclipse の全クライアントで共通。

仮置きの数値例
月額 $10,000 のワークロードで起きること
同じペイロードで平均 1.17× のトークンインフレが発生するとき、 キャッシュ/バッチを据え置いた場合の月額影響は概ね +$1,700（+17%）。 非英語主体のパイプラインでは +$3,500（+35%）まで跳ねる可能性がある、 というのが FinOps 系レビューの概算。
キャッシュ運用を詰めない限り、「価格表不変」の裏でひそかに請求が膨らむ。

割引スタックの組み方

システムプロンプトや長い参照ドキュメントは必ずキャッシュ境界より前に置く。 ヒットすると入力コストは最大 90% OFF。エージェントの中核ループでは 節約効果が最大化しやすい。
バッチ処理は 50% 割引。大量のドキュメント要約や分類は できるだけ同期呼び出しから引き剥がす。
Opus 4.7 の強みは計画・分解・検証といった上流工程。 抽出/整形/分類のような狭い実行は、より軽量・高速・安価なモデルに委譲。
冗長な英語指示は箇条書きや命令形に絞る。JSON は不要フィールドを削り、 コードは不要コメントを出力形式から除く。1.17× のインフレを 1.05× に 戻せれば、差はそのまま利益に変わる。
